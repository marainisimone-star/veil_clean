# Veil Vetrina — UX Flows (v1.0)

Data: 2026-02-09

Obiettivo: descrivere i flussi principali per l’esperienza utente della Vetrina.

## 1) Feed Vetrine
**Schermata: Vetrine**
- Lista pubblica di vetrine.
- Ogni card mostra: titolo, tema, tags.
- CTA: “Open showcase”.

**Azioni**
- Cerca per tema/tag.
- Crea nuova vetrina (sempre disponibile).

## 2) Dettaglio Vetrina (osservatore)
**Schermata: Vetrina**
- Header: titolo, tema, tags, regole core + linee guida.
- Timeline dei messaggi (pubblica).

**Azioni**
- Commenta (sei attivo subito).
- Salva (bookmark).
- Condividi link pubblico.

## 3) Partecipazione (accesso libero)
**Step**
1. L’utente entra e partecipa subito.
2. Se viola le regole → warning.
3. Recidiva → restricted → excluded.

## 4) Regole e materiali
**Schermata: Regole**
- Core rules (non modificabili).
- Regole custom del creator.
- Sezione “Materiali consigliati” (free/premium).
- Quiz opzionale (se previsto).

## 5) Moderazione AI (feedback)
**Flow**
- Se contenuto borderline → avviso e suggerimento di modifica.
- Se offensivo → blocco con messaggio chiaro.
- Visualizzazione limitata per i contenuti bloccati.

## 6) Vetrine figlie
**Suggerimento AI**
- Banner: “Questo thread è off-topic ma interessante. Vuoi creare una vetrina figlia?”
- Azioni: “Crea”, “Ignora”.

**Creazione manuale**
- Qualunque utente può creare una vetrina figlia.
- Le regole ereditano dalla vetrina madre, modificabili.

## 7) Inviti
**Azioni**
- Invita via link pubblico.
- Invita via email (esterni).

L’invito non garantisce accesso: valgono sempre le regole.

## 8) Segnali di qualità (no like)
- Bookmark/Save
- Highlight (qualità)
- Verified info (alto 5W)

## 9) Ranking (interno, non esposto in UI)
- Utilizzato solo per ordinare il feed.
- Nessun punteggio mostrato all’utente.
