
Data: 2026-02-09

Obiettivo: definire un MVP concreto per la parte Social “Vetrina”.

## Fase 0 (pre-MVP)
- Allineamento rules core (anti-abusi).
- Definizione schema dati (Firestore).

## MVP Fase 1 (2-3 settimane)
**Funzioni minime:**
- Feed vetrine pubbliche.
- Dettaglio vetrina (lettura + commento).
- Regole core + custom visibili.
- Accesso libero (no approvazione manuale).
- Moderazione AI base (flag e warning → restricted → excluded).
- Menu regole standard + linee guida non vincolanti.
- 5W usato per evidenziare mancanze (non per giudicare sentiment).

**UI**
- Tab “Vetrine” visibile in app.
- Lista + dettaglio + modale regole.

**Back-end**
- Firestore: vetrine, messages, participants.
- Rules core in codice + flags AI mock.

## MVP Fase 2 (3-4 settimane)
**Espansioni:**
- Vetrine figlie suggerite dall’AI.
- Materiali gratuiti e premium.
- Quiz opzionali.
- Inviti da chiunque.

## MVP Fase 3 (4+ settimane)
**Monetizzazione:**
- Sponsorizzazioni.
- Highlight ufficiali.
- Badge reputazione informativa.

## KPI iniziali
1. Numero vetrine create.
2. Numero interventi moderazione AI.
3. Tempo medio fino a warning.
